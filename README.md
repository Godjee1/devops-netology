2. Изучение ТЗ всей командой
3. Определение модели участия клиента в проекте, при участии DevOps инженера и разработчиков

5. Старт разработки проекта
6. Организация процесса разработки (devops)
7. Создание основной ветки репозитория для конечного продукта (devops)
8. Предоставление ветки для разработки и тестирования
9. Обеспечение бекапа всех веток (devops)
10. Обеспечение возможности отката к предыдущим версиям продукта (devops)
11. Разработка продукта (Разарботчики)
12. Тестирование коммитов (Тестеровщики)
13. Загрузка коммитов, успешно пройденных тестирование в основную ветку (devops)
14. Демонстарция результата клиенту по основной ветке (менеджер)
15. Успешное завершение проекта по достижению поставленной задачи
